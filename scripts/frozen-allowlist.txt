# Frozen-string allowlist for the retired "planctl" AND "agentwrap" names.
#
# "planctl" (fn-889) retired to "keeper plan" / ".keeper" / "plan", with two
# narrow classes of literal PERMANENTLY frozen (git-history trailers + src/db.ts
# schema-history) — a PROGRESSIVE guard that pins only those.
#
# "agentwrap" (fn-1018 + fn-1020) retired to ZERO: env vars -> KEEPER_AGENT_*,
# config dir -> ~/.config/keeper, runtime state dir -> ~/.local/state/keeper-agent.
# scripts/lint-retired-name.sh Check C enforces zero "agentwrap" repo-wide; the
# ONLY records this file still carries for it are the count-pins on the two
# state-dir relocation files that must name their pre-relocation path forever.
#
# "keeper pair" (fn-1032) retired to ZERO: the `keeper pair` CLI folded onto
# `keeper agent` (`agent run` blocking single-shot + `agent panel start|wait`
# detached/panel fan-out). scripts/lint-retired-name.sh Check D enforces zero
# space-separated "keeper pair" repo-wide (the colon-separated `keeper:pair` SKILL
# name is a live capability and never matches). This file carries NO records for
# it — the check is a pure hardcoded grep-clean with no count-pinned residue.
#
# This file is the single source of truth the guard (scripts/lint-retired-name.sh)
# reads to tell a frozen literal from a renamable symbol.
#
# Format: pipe-delimited records, one per line. Blank lines and `#` comments are
# ignored. Four record kinds:
#
#   anchor|<relpath>|<exact-substring>
#       A frozen literal that must appear verbatim in <relpath>. The guard
#       fails if the substring is missing (a sweep clobbered a frozen literal).
#       The substring is matched as a fixed string (grep -F), so regex/quoting
#       metacharacters are literal.
#
#   count|<relpath>|<n>[|<token>]
#       A PURE-frozen file: every case-insensitive <token> occurrence in it is a
#       frozen literal (schema-history, trailer, or a permanent relocation path).
#       <token> defaults to "planctl" when the 4th field is omitted. The guard
#       fails if the live count != <n> — catching both an accidental clobber
#       (count drops) and a planted retired-name edit (count rises).
#
#   forbid|<relpath>|<substring>
#       A cleaned file pinned to ZERO occurrences of <substring>: the guard
#       fails if the fixed-string, case-insensitive substring appears anywhere
#       in <relpath> (a retired name regrew into a purged surface). SINGLE FILE
#       only — never a directory walk. A missing target file = FAIL.
#
#   exempt|<relpath>
#       A file that legitimately still carries un-renamed "planctl" references
#       the guard must NOT flag, but whose tokens are NOT all frozen (mixed
#       frozen + renamable). Listed for documentation; the guard does not pin a
#       count on these. As .6/.7/.8 land, each sweep removes its file from here.
#

# them without orphaning every historical commit's plan attribution.

### Approach

Port jobctl's per-turn parser from `/Users/mike/code/arthack/apps/cli_common/cli_common/subagent_invocations.py` to a TypeScript module `src/subagent-invocations.ts`. The TS module is pure — no DB writes, no I/O, no third-party deps. It exports the per-event reducer helpers task .3 will call. Trim captured fields to the v1 set: `ts`, `tool_use_id`, `agent_id`, `turn_seq`, `subagent_type`, `description`, `prompt_chars`, `status`, `duration_ms`. Drop `tokens` and `tool_use_count` (billing-flavored — usagectl territory, not keeper).

Module exports:
- `extractTurnSeq(db, jobId, agentId): number` — `SELECT COALESCE(MAX(turn_seq), -1) + 1 FROM subagent_invocations WHERE job_id = ? AND agent_id = ?`. Pure function of persisted rows; re-fold safe. Returns 0 on a fresh agent_id.
- `findOpenTurnForStop(db, jobId, agentId): number | null` — `SELECT turn_seq FROM subagent_invocations WHERE job_id = ? AND agent_id = ? AND duration_ms IS NULL ORDER BY turn_seq DESC LIMIT 1`. Gates on `duration_ms IS NULL` ALONE per Python's fn-480 comment (PostToolUse:Agent can flip status to 'ok' before SubagentStop arrives for Task calls).
- `findBridgePreToolUse(db, sessionId, toolUseId): {description: string|null, prompt_chars: number, subagent_type: string|null} | null` — looks up the matching PreToolUse:Agent row from events: `SELECT data FROM events WHERE session_id = ? AND tool_use_id = ? AND hook_event = 'PreToolUse' AND tool_name = 'Agent' ORDER BY id ASC LIMIT 1`. Parses `data.tool_input.description`, `data.tool_input.prompt` (length only — never the body), `data.tool_input.subagent_type`. Returns null when no row matches. Cross-job-safe: `session_id` is mandatory in WHERE.
- `resolveBridgeAgentId(event): string | null` — reads `event.subagent_agent_id` first; falls back to parsing `event.data.tool_response.agentId` (defense-in-depth for pre-fn-390 historical rows). Returns null if neither resolves.
- `truncateDescription(s, maxChars=200): string` — port of Python's `_DESCRIPTION_MAX_CHARS=200`. Constant exported from this module as the single source of truth.
- `canonicalizeRow(row): string` — produces the wire-canonical JSON shape for golden-fixture comparison. Matches Python's `json.dumps(sort_keys=True, separators=(',', ':'))` byte-for-byte (keys alphabetically sorted, no whitespace between separators, `null` for missing fields, integers without trailing `.0`).

Golden fixture at `test/fixtures/subagent_invocation_cases.jsonl` — generated by `scripts/generate-subagent-invocation-fixtures.py`. The Python script imports the upstream `subagent_invocations.py:parse_rows`, drives it over hand-crafted event sequences, and writes one JSONL line per (input-sequence, expected-output-rows) pair. The TS test reads the fixture, feeds inputs through the per-event helpers (against a fresh in-memory DB seeded with task .1's schema), and asserts byte-identical canonical JSON output against expected.

Edge cases as named fixture entries:
- `clean-close`: Pre + Start + Stop + Post (status='ok', all fields populated)
- `still-running`: Pre + Start (status='running', duration_ms=null)
- `orphan-stop`: SubagentStop with no matching open Start (drop, no row written)
- `orphan-failure`: PostToolUseFailure:Agent (drop, no row mutation)
- `post-before-stop`: PostToolUse:Agent arrives before SubagentStop (Anthropic-confirmed Task ordering; status flips to 'ok' before SubagentStop closes the row; SubagentStop gates on `duration_ms IS NULL` not `status`)
- `multi-turn`: same agent_id, two SubagentStart/Stop pairs (turn_seq increments per agent_id)
- `interleaved`: two distinct agents' events interleaved in one session
- `pre-without-post`: PreToolUse:Agent + SubagentStart + SubagentStop without PostToolUse — description/prompt_chars stay unset on the turn-0 row (matches Python's bridge-fold-never-fires behavior)
- `cross-job-tool-use-id-collision`: two sessions with same tool_use_id (verify the session_id WHERE prevents contamination)

Fixture regeneration is EXPLICIT script call only — never auto-regenerate in CI. The generator script's invocation is documented in its docstring:

    python3 scripts/generate-subagent-invocation-fixtures.py > test/fixtures/subagent_invocation_cases.jsonl

### Risks

- **Byte-identical parity across Python/TS JSON serialization.** Python's `json.dumps(sort_keys=True, separators=(',', ':'))` has specific byte output: keys alphabetically sorted, no whitespace between separators, integers without trailing `.0`. The TS canonicalizer must match exactly. Test the canonicalizer in isolation against hand-crafted dicts before trusting it on parser output.
- **null vs undefined.** SQLite NULLs come back as `null` in bun:sqlite. Python's `None` maps to `null`. `JSON.stringify` DROPS `undefined` fields — the canonicalizer must emit explicit `null` for missing fields. Check this in a unit test.
- **No in-process buffer in keeper.** The Python parser uses `pending_pre_by_tool_use_id` map during single-pass walk. The TS port uses fold-time events-table lookup INSTEAD — no buffer. Verify no Python state leaks into the TS port that implicitly assumes the buffer exists.
- **Description truncation constant.** Python's `_DESCRIPTION_MAX_CHARS=200` is the source of truth. The TS constant must match exactly. Export it from `src/subagent-invocations.ts`; reference Python's value in a comment.
- **Cross-job isolation.** `findBridgePreToolUse` MUST include `session_id` in WHERE. Two sessions can interleave with identical `tool_use_id` strings (Anthropic-unique but not enforced cross-session in our DB). The `cross-job-tool-use-id-collision` fixture entry verifies this.
